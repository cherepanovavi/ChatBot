				<div id="mw-content-text" lang="ru" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>SDRAM</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a>&#160;<span lang="en" style="font-style:italic;"><b>S</b>ynchronous <b>D</b>ynamic <b>R</b>andom <b>A</b>ccess <b>M</b>emory</span>&#160;— синхронная динамическая память с произвольным доступом)&#160;— тип <a href="/wiki/%D0%97%D0%B0%D0%BF%D0%BE%D0%BC%D0%B8%D0%BD%D0%B0%D1%8E%D1%89%D0%B5%D0%B5_%D1%83%D1%81%D1%82%D1%80%D0%BE%D0%B9%D1%81%D1%82%D0%B2%D0%BE_%D1%81_%D0%BF%D1%80%D0%BE%D0%B8%D0%B7%D0%B2%D0%BE%D0%BB%D1%8C%D0%BD%D1%8B%D0%BC_%D0%B4%D0%BE%D1%81%D1%82%D1%83%D0%BF%D0%BE%D0%BC" title="Запоминающее устройство с произвольным доступом">запоминающего устройства</a>, использующегося в <a href="/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80" title="Компьютер">компьютере</a> и других цифровых устройствах в качестве <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%B8%D0%B2%D0%BD%D0%B0%D1%8F_%D0%BF%D0%B0%D0%BC%D1%8F%D1%82%D1%8C" title="Оперативная память">ОЗУ</a>.
</p><p>В отличие от других типов DRAM, использовавших <a href="/w/index.php?title=%D0%90%D1%81%D0%B8%D0%BD%D1%85%D1%80%D0%BE%D0%BD%D0%BD%D1%8B%D0%B9_%D0%BE%D0%B1%D0%BC%D0%B5%D0%BD&amp;action=edit&amp;redlink=1" class="new" title="Асинхронный обмен (страница отсутствует)">асинхронный обмен</a> данными, ответ на поступивший в устройство управляющий сигнал возвращается не сразу, а лишь при получении следующего <a href="/wiki/%D0%A2%D0%B0%D0%BA%D1%82%D0%BE%D0%B2%D1%8B%D0%B9_%D1%81%D0%B8%D0%B3%D0%BD%D0%B0%D0%BB" title="Тактовый сигнал">тактового сигнала</a>. Тактовые сигналы позволяют организовать работу SDRAM в виде <a href="/wiki/%D0%9A%D0%BE%D0%BD%D0%B5%D1%87%D0%BD%D1%8B%D0%B9_%D0%B0%D0%B2%D1%82%D0%BE%D0%BC%D0%B0%D1%82" title="Конечный автомат">конечного автомата</a>, исполняющего входящие команды. При этом входящие команды могут поступать в виде непрерывного потока, не дожидаясь, пока будет завершено выполнение предыдущих инструкций (<a href="/wiki/%D0%92%D1%8B%D1%87%D0%B8%D1%81%D0%BB%D0%B8%D1%82%D0%B5%D0%BB%D1%8C%D0%BD%D1%8B%D0%B9_%D0%BA%D0%BE%D0%BD%D0%B2%D0%B5%D0%B9%D0%B5%D1%80" title="Вычислительный конвейер">конвейерная обработка</a>): сразу после команды записи может поступить следующая команда, не ожидая, когда данные окажутся записаны. Поступление команды чтения приведёт к тому, что на выходе данные появятся спустя некоторое количество тактов&#160;— это время называется <a href="/wiki/%D0%9B%D0%B0%D1%82%D0%B5%D0%BD%D1%82%D0%BD%D0%BE%D1%81%D1%82%D1%8C_(%D1%85%D0%B0%D1%80%D0%B0%D0%BA%D1%82%D0%B5%D1%80%D0%B8%D1%81%D1%82%D0%B8%D0%BA%D0%B0_%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%82%D0%B8%D0%B2%D0%BD%D0%BE%D0%B9_%D0%BF%D0%B0%D0%BC%D1%8F%D1%82%D0%B8)" class="mw-redirect" title="Латентность (характеристика оперативной памяти)">задержкой</a> и является одной из важных характеристик данного типа устройств.
</p><p>Циклы обновления выполняются сразу для целой строки, в отличие от предыдущих типов <a href="/wiki/DRAM" title="DRAM">DRAM</a>, обновлявших данные по внутреннему счётчику, используя способ обновления по команде CAS перед RAS.
</p>

<h2><span id=".D0.98.D1.81.D1.82.D0.BE.D1.80.D0.B8.D1.8F_.D0.B8.D1.81.D0.BF.D0.BE.D0.BB.D1.8C.D0.B7.D0.BE.D0.B2.D0.B0.D0.BD.D0.B8.D1.8F"></span><span class="mw-headline" id="История_использования">История использования</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SDRAM&amp;veaction=edit&amp;section=1" class="mw-editsection-visualeditor" title="Редактировать раздел «История использования»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=SDRAM&amp;action=edit&amp;section=1" title="Редактировать раздел «История использования»">править код</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Массовый выпуск SDRAM начался в 1993 году. Первоначально этот тип памяти предлагался в качестве альтернативы для дорогой видеопамяти (<a href="/wiki/VRAM" title="VRAM">VRAM</a>), однако вскоре SDRAM завоевал популярность и стал применяться в качестве ОЗУ, постепенно вытесняя другие типы динамической памяти. Последовавшие затем технологии DDR позволили сделать SDRAM ещё эффективнее. За разработкой <a href="/wiki/DDR_SDRAM" title="DDR SDRAM">DDR SDRAM</a> последовали стандарты <a href="/wiki/DDR2_SDRAM" title="DDR2 SDRAM">DDR2 SDRAM</a>, <a href="/wiki/DDR3_SDRAM" title="DDR3 SDRAM">DDR3 SDRAM</a> и <a href="/wiki/DDR4_SDRAM" title="DDR4 SDRAM">DDR4 SDRAM</a>.
</p>
<h2><span class="mw-headline" id="SDR_SDRAM">SDR SDRAM</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=SDRAM&amp;veaction=edit&amp;section=2" class="mw-editsection-visualeditor" title="Редактировать раздел «SDR SDRAM»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=SDRAM&amp;action=edit&amp;section=2" title="Редактировать раздел «SDR SDRAM»">править код</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Первый стандарт SDRAM с появлением последующих стандартов стал именоваться SDR (Single Data Rate&#160;— в отличие от Double Data Rate). За один такт принималась одна управляющая команда и передавалось одно слово данных. Типичными тактовыми частотами были 66, 100 и 133 МГц. Микросхемы SDRAM выпускались с <a href="/wiki/%D0%A8%D0%B8%D0%BD%D0%B0_%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85" title="Шина данных">шинами данных</a> различной ширины (обычно 4, 8 или 16 бит), но как правило, эти микросхемы входили в состав 168-контактного модуля <a href="/wiki/DIMM" title="DIMM">DIMM</a>, который позволял прочитать или записать 64 бита (в варианте без <a href="/wiki/ECC" class="mw-redirect" title="ECC">контроля чётности</a>) или 72 бита (с контролем чётности) за один такт.
</p>
